    # 介绍

    Vue

    Vue 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

    # v-text和v-html

    我们已经用的是,这种情况是有弊端的，就是当我们网速很慢或者javascript出错时，会暴露我们的。
    Vue给我们提供的v-text,就是解决这个问题的。
    双大括号和v-text会将数据解释为纯文本，而非HTML。为了输出真正的HTML，你就需要使用v-html 指令。
    需要注意的是：在生产环境中动态渲染HTML是非常危险的，因为容易导致XSS攻击。所以只能在可信的内容上使用v-html，永远不要在用户提交和可操作的网页上使用。

    代码:

    <div v-text="htmlmsg"> {{msg}}</div> <!--输出 <span>这里是span</span>-->
<div v-html="htmlmsg">{{msg}}</div> <!--输出 这里是span-->
<!--两者皆覆盖{{msg}} -->
<script>
var app=new Vue({
        el:"#app",
        data:{
            msg: "Hello World",
            htmlmsg: "<span>这里是span</span>"
        }
});
</script>

    # v-if、v-else、v-show、

    v-if、v-else懂得都懂if else嘛，传递一个布尔类型去进行判断是否展示元素。

    v-show等同于display:none 推荐在数据量比较大，控制显示状态切换频繁的情况下使用。

    <div v-show="flag">Hello World</div>
<div v-if="flag_if">Hello World</div>
    1
    2

    两者区别

    v-if:直接操作dom元素，通过对dom元素的删除和添加来展示和隐藏，判断是否加载，可以减轻服务器的压力，在需要时加载。但数据量过大时，频繁添加删除dom元素会卡顿。

    v-show:前文提过等同于display:none，存在于页面中，判断为true时展示,可以使客户端操作更加流畅。

    排序

    我们已经顺利的输出了我们定义的数组，但是我需要在输出之前给数组排个序。这里我采用computed:属性。

     <div id="app">
       <h2 v-for="item,index in items">index:{{index}} item: {{item}}</h2>
        <h2>排序后：{{sortItems}}</h2>
<div>
<script>
    const app = new Vue({
        el: "#app",
        data: {
            items: [6, 12, 45, 64, 18, 8],
        },
        computed: {//计算属性
            sortItems: function () {
                return this.items.sort();
            },
    });
</script>

    我们在computed里新声明了一个对象sortItems，如果不重新声明会污染原来的数据源，这是Vue不允许的，所以要重新声明一个对象。

    顺利运行，但是这段代码程序还是有个小Bug，输出结果为:

    排序后: [ 12, 18, 45, 6, 64, 8 ]

    因为它本身的排序是根据数字的第一位来判断，所以自己编写一个方法sortNumber，然后传给sortItems函数解决这个Bug。

     <div id="app">
       <h2 v-for="item,index in items">index:{{index}} item: {{item}}</h2>
        <h2>排序后：{{sortItems}}</h2>
<div>
<script>
    const app = new Vue({
        el: "#app",
        data: {
            items: [6, 12, 45, 64, 18, 8],
        },
        computed: {//计算属性
            sortItems: function () {
                return this.items.sort(sortNumber);
            },
    });
    function sortNumber(a, b) {//注意，这个函数是写在vue实例外边的
        return a - b
    }
</script>

    维护状态
    当 Vue 正在更新使用 v-for 渲染的元素列表时，它默认使用“就地更新”的策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新每个元素，并且确保它们在每个索引位置正确渲染。这个类似 Vue 1.x 的 track-by="$index"。
    这个默认的模式是高效的，但是只适用于不依赖子组件状态或临时 DOM 状态 (例如：表单输入值) 的列表渲染输出。
    为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一 key attribute。
    也就是遍历对象数组最好给每个对象添加个唯一键值：

    <div v-for="item in items" v-bind:key="item.id">
  <!-- 内容 -->
</div>
    1
    2
    3

    建议尽可能在使用 v-for 时提供 key attribute，除非遍历输出的 DOM 内容非常简单，或者是刻意依赖默认行为以获取性能上的提升。
    因为它是 Vue 识别节点的一个通用机制，key 并不仅与 v-for 特别关联。后面我们将在指南中看到，它还具有其它用途。
    引用 (opens new window)

    顺带一提，不推荐同时使用 v-if 和 v-for。请查阅风格指南 (opens new window)以获取更多信息。 当 v-if 与 v-for 一起使用时，v-for 具有比 v-if 更高的优先级。

    # 生命周期

    废话不多说，先上图 

     <div id="app">
       {{msg}}
       <p><button @click="add">+1</button></p>
   </div>
       <button onclick="app.$destroy()">销毁</button>

   <script type="text/javascript">
       var app=new Vue({
           el:'#app',
           data:{
               msg:1
           },
           methods:{
               add:function(){
                   this.msg ++;
               }
           },
           beforeCreate:function(){
               console.log('1-beforeCreate 初始化之后');
               //第一步初始化，无法访问data中的数据、methods中的方法。
               //因为生命周期钩子、事件、代理数据还未开始初始化。
           },
           created:function(){
               console.log('2-created 创建完成');
               //完成数据代理，可以访问data中的数据、methods中的方法。
               //一般用于网页的加载画面，比如此博客的吃豆人动画
           },
           beforeMount:function(){
               console.log('3-beforeMount 挂载之前');
               //对DOM节点的操作尚且不能生效，因为还没有挂载(mount)。
           },
           mounted:function(){
               console.log('4-mounted 被创建');
               //挂载完毕，此时可以操作DOM节点，此时组件已脱离初始化，进入了运行阶段
               //一般在此阶段：开启定时器、发送网络请求等准备性工作，还有结束加载动画。
               
           },
           beforeUpdate:function(){
               console.log('5-beforeUpdate 数据更新前');
               //在此钩子中：数据是新的但页面是旧的
               //例如我在此处运行add()单击事件，msg++，但数据不会更新到网页中。
           },
           updated:function(){
               console.log('6-updated 被更新后');
               //页面和数据保持同步
           },
           activated:function(){
               console.log('7-activated');
           },
           deactivated:function(){
               console.log('8-deactivated');
           },
           beforeDestroy:function(){
               console.log('9-beforeDestroy 销毁之前');
               //此钩子中：所有的data、methods、指令等等，都处于可用状态，马上要执行销毁过程
               //一般在此钩子中：关闭定时器、取消订阅消息等收尾工作
              
           },
           destroyed:function(){
               console.log('10-destroyed 销毁之后')
               //移除所有数据的监视，不会更新页面了(但是东西还在)。
               //有点鸡肋，相当于我人都已经死了，发烂发臭，躺棺材里了，你还要我怎样，去找林正英玩耍咩？
           }

       })
   </script>

    # 自定义指令 Vue.directive

    # 官方文档用法

    除了核心功能默认内置的指令 (v-model 和 v-show)，Vue 也允许注册自定义指令。注意，在 Vue2.0 中，代码复用和抽象的主要形式是组件。然而，有的情况下，你仍然需要对普通 DOM 元素进行底层操作，这时候就会用到自定义指令。举个聚焦输入框的例子，如下：

    / 注册一个全局自定义指令 `v-focus`
Vue.directive('focus', {
 // 当被绑定的元素插入到 DOM 中时……
 inserted: function (el) {
   // 聚焦元素
   el.focus()
 }
})

    # 自定义指令传的3个参数

    • el: 指令所绑定的元素，可以用来直接操作DOM。
    • binding: 一个对象，包含指令的很多信息。

    # 自定义指令的生命周期

    自定义指令有五个生命周期（也叫钩子函数），分别是 bind,inserted,update,componentUpdated,unbind

    1. bind:只调用一次，指令第一次绑定到元素时调用，用这个钩子函数可以定义一个绑定时执行一次的初始化动作。
    2. inserted:被绑定元素插入父节点时调用（父节点存在即可调用，不必存在于document中）。
    3. update:被绑定于元素所在的模板更新时调用，而无论绑定值是否变化。通过比较更新前后的绑定值，可以忽略不必要的模板更新。
    4. componentUpdated:被绑定元素所在模板完成一次更新周期时调用。
    5. unbind:只调用一次，指令与元素解绑时调用。
    bind:function(){//被绑定
    console.log('1 - bind');
},
inserted:function(){//绑定到节点
     console.log('2 - inserted');
},
update:function(){//组件更新
     console.log('3 - update');
},
componentUpdated:function(){//组件更新完成
     console.log('4 - componentUpdated');
},
unbind:function(){//解绑
     console.log('1 - bind');
}

    const ttt={}

ttt.install=function(Vue,options){

   //把全局自定义指令放在这块
   Vue.directive('text-upper',function(el,binding){
       el.innerText = binding.value.toUpperCase()
   })
   Vue.directive('text-lower',function(el,binding){
       el.innerText = binding.value.toLowerCase()
   })
   //给Vue自身添加属性
   Vue.projectName='学生管理系统'
   Vue.testVersion='v1.0.1'
   Vue.showInfo=function () {
       console.log('获取一些数据');
     }
   //给Vue原型上添加数据
   Vue.prototype.$random=function(min,max){//获取一个区间的随机数
       return Math.floor( Math.random()*(max-min)+min)
   }
}
