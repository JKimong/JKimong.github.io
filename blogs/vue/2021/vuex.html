    Vuex

    a salt fish 7/1/2021 vue

    Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。Vuex 也集成到 Vue 的官方调试工具 devtools extension (opens new window)，提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。
    github站点：https://github.com/vuejs/vuex
    在线文档：https://vuex.vuejs.org/zh/

    # 什么是“状态管理模式”？

    让我们从一个简单的 Vue 计数应用开始：

    new Vue({
  // state
  data () {
    return {
      count: 0
    }
  },
  // view
  template: `
    <div>{{ count }}</div>
  `,
  // actions
  methods: {
    increment () {
      this.count++
    }
  }
})

    # 什么情况下我应该使用 Vuex？

    1. 多个组件依赖于同一个状态
    2. 来自不同组件的行为需要变更为同一状态
    3. 多个组件要共享状态 Vuex 可以帮助我们管理共享状态，并附带了更多的概念和框架。这需要对短期和长期效益进行权衡。

    如果您不打算开发大型单页应用，使用 Vuex 可能是繁琐冗余的。确实是如此——如果您的应用够简单，您最好不要使用 Vuex。一个简单的store 模式 (opens new window)就足够您所需了。但是，如果您需要构建一个中大型单页应用，您很可能会考虑如何更好地在组件外部管理状态，Vuex 将会成为自然而然的选择。引用 Redux 的作者 Dan Abramov 的话说就是：

    Flux 架构就像眼镜：您自会知道什么时候需要它。

    # Vuex原理图

    图解:

    • Backend API:在这个环节可以发送网络请求。
    • Devtools:Vuex的官方调试工具。
    • State:一个对象，里面是我们定义的一些初始值。
      ↓Render:渲染
    • Vue components:自己的Vue组件。
      ↓ Dispatch:分发
    • Actions:调用对应的动作，Actions其实就是个中转站，本质是个对象{组件函数名:function}但是不使用它，相当于餐厅中的服务员。 ↓ Commit:提交
    • Mutaions:用于加工状态，本质也是个对象，在里面处理传过来的参数和函数，使其发生化学反应ԅ(¯﹃¯ԅ)。
      ↓ Mutate:改变状态(加工)
    • State:接收处理完的数据。
      ↓ Render:渲染
    • Vue components:通过组件显示定义的各种变量之类的。

    # Vuex求和案例

    老规矩，通过一个案例来说明讲解vuex的使用。
    做一个简单的需求实现，但是数据和函数由一个store.js文件来管理。
    首先，在src目录下新建一个文件夹vuex，里面新建一个文件store.js（也有src/store/index.js这个写法）。

    # store.js和main.js

    //引入Vue
import Vue from 'vue'
//引入Vuex
import Vuex from 'vuex';
// 应用插件
Vue.use(Vuex)
//初始化应用状态，要写成一个对象，因为该state要管理n多个组件的状态
const state= {
    sum:0,
    name:'李狗蛋'
    
}
//创建一个actions,值为一个对象，包含：n个响应组件“动作”的函数
const actions={
    //自带一个参数对象，迷你版的$store，我们一般称它为context
    add(context,value){//我要用context.commmit()去通知mutations加工状态
        context.commit('ADD',value)
        //把
    },
    reduction(context,value){
        context.commit('REDUCTION',value)
    },
    addOdd(context,value){
        if(state.sum % 2){
            context.commit('ADDODD',value)
        }  
    },
    addAsync(context,value){
        setTimeout(() => {
            context.commit('ADDASYNC',value)
        }, 500);
    }
}

const mutations={
    ADD(state,value){
        state.sum+=value;
    },
    REDUCTION(state,value){
        state.sum-=value;
    },
    ADDODD(state,value){
        state.sum+=value;
    },
    ADDASYNC(state,value){
        state.sum+=value;
    }

}

//getters中配置的是state中的数据经过加工后的值
const getters={
    //假设这是一个需要经过复杂计算出值的函数
    bigSum(state){
        return state.sum*100
    }
    //类似于计算属性computed
}

//创建store，用于管理：state对象、actions对象、mutations对象
const store = new Vuex.Store({
    state,
    actions,
    mutations,
    getters
})
//暴露出去
export default store

    # Count.vue组件

    App.vue就省略一下，在这个组件里没什么新的知识点。

    <template>
  <div>
      <h2>当前求和为&nbsp;：{{sum}}</h2>
      <h2>计算后的值&nbsp;：{{$store.getters.bigSum}}</h2>
      <h2>state中的name：{{name}}</h2>
      <select v-model.number="n">
          <option value="1">1</option>
          <option value="2">2</option>
          <option value="3">3</option>
      </select>&nbsp;
    <button @click="increment">+</button>&nbsp;
    <button @click="decrement">-</button>&nbsp;
    <button @click="incrementOdd">奇数再加</button>&nbsp;
    <button @click="incrementAsync">异步加</button>&nbsp;
  </div>
</template>

<script>
import {mapState} from'vuex'
export default {
    name:'Count',
    data(){
        return{
            
            n:1
        }
    },methods:{
        increment(){
            this.$store.dispatch('add',this.n)
            // this.$store.dispatch('绑定函数名',传过去一个参数)
        },
        decrement(){
            this.$store.dispatch('reduction',this.n)
        },
        incrementOdd(){
             this.$store.dispatch('addOdd',this.n)
        },
        incrementAsync(){
             this.$store.dispatch('addAsync',this.n)
        }
    },
    computed:{
        ...mapState(['sum','name'])
        /*
        上面的一句代码等同于：
        sum(){
            return this.$store.sum
        }
        name(){
            return this.$store.name
        }
        */
       //mapGetters同理。
       //...mapGetters(['bigSum'])
    }
}
</script>

    两者区别(个人理解)：

    其实就是分发Ations和Mutations到组件中，所以两者区别具体看Actions和Mutions的区别

    • Actions：能够使用异步方法，无法更新state。
    • Mutions：无法使用异步，可以更新state。
      因为这两者的特性，所以默认在Actions里写函数方法，然后把方法commit传去Mutations中更新state，也就是案例中的写法。
